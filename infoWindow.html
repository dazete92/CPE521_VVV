    What is a Cookie? (click)
    A cookie is a piece of data sent from a website that is stored in the browser.

    What are cookies used for (click)
    Cookies are used to remember browser state information, such as browsing activity and passwords. Each time a website is loaded, the browser sends the cookie back to the website to remind it of previous activity. One popular example that uses cookies is an online shopping cart, where items remain in the cart when the online store is revisited. Another popular example is login preservation, which allows a user to visit their email page without having to log in every time.

    Security Implications of Cookies (click)
    If cookies aren't secured properly or stolen, they can be used by an attacker to steal information or impersonate users. If a valid user is using cookies to authenticate access to their email and an attacker steals their cookies, the attacker can then access the user's email without needing to provide login information.

    Best Practices (click)

    • Understand how to properly use cookies. This includes how to use them, what they are used for, and why they are used.
    • Understand cookie attributes. This includes what attributes are available, what each attribute does, how to set them, and when to set them. See the table below.
    • Always encrypt or encode cookies that contain sensitive information
    • Test and verify that the cookie attributes are correct

    Important Cookie Attributes
    Attribute Description
    Secure Tells the browser to only send the cookie over a secure connection (HTTPS). Enabling this option prevents a cookie from being tranmitted through unencrypted connections.
    HttpOnly Tells the browser to only allow HTTP requests to access the cookie's information. This prevents access from any JavaScript code. Not supported on all browsers.
    Domain Tells the browser to check the URL being requested against the domain name of the server that is processing the request.
    Path Specifies the URLs where the cookie is considered valid. Used in conjunction with the domain attribute to verify the validity of a request.
    Expires Specifies the expiration date of the cookie.

    General XSS Information (click)
    What is XSS? (click)
    Cross site scripting (XSS) is a type of code injection, in which malicious code (usually javascript) is inserted into an otherwise safe, trusted web site. When this code gets loaded into the browser of another user, it is executed. For example, a malicious user may enter <script>alert("xss");</script> into the comment field of a forum which saves it as a comment on the server. When another user visits the forum and loads the web page their browser, believing the code to be from the trusted website, will execute the script.

    Security Implications of XSS (click)
    XSS can cause a variety of problems for the end user that range in severity from an annoyance to complete account compromise. The most severe XSS attacks involve disclosure of the user"s session cookie, allowing an attacker to hijack the user"s session and take over the account. Other damaging attacks include the disclosure of end user files, installation of Trojan horse programs, redirect the user to some other page or site, or modify presentation of content.
